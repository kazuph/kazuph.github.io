Liquid DOM は、CSS の glassmorphism ではなく、WebGPU の renderer と DOM-backed な scene graph で液体ガラスを描く実験的な UI ライブラリです。このページではサイト全体のレイアウトは変えず、この記事が表示された場合だけ @liquid-dom/core@0.1.0 を読み込み、下のデモ領域だけで Liquid DOM を起動しています。

このページで試すもの

まず背景となる普通の DOM を置き、その上に Liquid DOM の Scene、Container、Glass、Html を重ねます。ガラスの見た目は CSS では作らず、Liquid DOM の Renderer が生成する <canvas layoutsubtree> に描かせます。

3つのデザインサンプル

Reading card

文章を載せるガラス。背景が動いても、内容は Html node として glass の中に置ける。

Floating control

再生、速度、モード切り替えのような小さな操作面。Liquid DOM 側の pointer event も扱える。

Status lens

WebGPU renderer の状態や DOM capture の可否を表示する薄いレイヤー。今回のデモでは status badge がそれ。

CSSで捏造しないための境界線

このページにも配置や余白のための CSS はあります。ただし、ガラス表現そのものを CSS の backdrop-filter や半透明背景で代替していません。Liquid DOM の import、scene graph の生成、Renderer.render() の frame loop が成功したときだけ、デモ領域に Liquid DOM の canvas が出ます。HTML-in-Canvas がない環境では DOM-backed Html は追加せず、Liquid DOM の Glass shape だけを描画します。 

import { Container, Glass, Html, Renderer, Scene } from 'https://esm.sh/@liquid-dom/core@0.1.0?bundle'

Liquid DOM の構成

@liquid-dom/core は DOM-backed liquid-glass renderer です。低レベルの scene graph と WebGPU renderer を持ち、React を使わなくても Scene に Html や Container を追加して描画できます。React から使う場合は @liquid-dom/react、Three.js 連携には @liquid-dom/three や @liquid-dom/r3f が用意されています。

必要な環境

Liquid DOM の rendering には WebGPU が必要です。さらに DOM-backed な Html content を正しく GPU texture としてコピーするには、Chrome の実験的な HTML-in-Canvas / Canvas Draw Element API が必要です。対応していない環境では、通常 DOM の記事本文はそのまま読めますが、デモ領域の status は active-glass-only、webgpu-unavailable、render-error のいずれかになります。
